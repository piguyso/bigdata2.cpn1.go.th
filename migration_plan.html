แผนย้ายข้อมูลระบบบริหารและพัฒนาบุคลากร

จากระบบเดิม (Custom PHP Website) สู่ระบบใหม่ (Laravel 11 + Vite + Tailwind CSS)
ของศูนย์พัฒนาครูและบุคลากรทางการศึกษา สำนักงานเขตพื้นที่การศึกษาประถมศึกษาชุมพร เขต 1 (EE.CPN1)

เมนูสิทธิบัตร

ภาพรวมโครงสร้างของระบบเก่าใน /e

ระบบพัฒนาเดิมที่อยู่ภายในห้องไดเรกทอรี /e ทำหน้าที่เป็นระบบจัดเก็บข้อมูลประวัติครู (Teacher Profiles) ควบคู่ไปกับระบบจัดการคอร์สอบรมออนไลน์ (LMS) ข้อมูลระบบเก่าประกอบด้วยโมดูลสำคัญดังนี้:

โมดูลสำรวจและรายงานข้อมูลครู (Teacher Survey)

เก็บรวบรวมประวัติส่วนตัว, ประวัติการศึกษา, วิชาที่รับผิดชอบสอน, ผลงาน/รางวัล, และผลการทดสอบภาษาอังกฤษ (CEFR) และภาษาจีน (HSK) ของคุณครูในพื้นที่

โมดูลการเรียนออนไลน์ (LMS System)

ประกอบด้วยหน้าการเรียนรู้รายวิชา การบันทึกเวลาเรียนวิดีโอ การตรวจงานจากนักเรียน การทำข้อสอบวัดผลสัมฤทธิ์ก่อน-หลังเรียน และการออกใบประกาศนียบัตรอัจฉริยะ

โครงสร้างฐานข้อมูลที่พบในซอร์สโค้ด

ประเภทข้อมูล ตารางระบบเดิม (MySQL) บทบาทและรายละเอียด
แกนกลางระบบ users, system_announcements, banners ผู้ใช้งานระบบ แผงประกาศข่าว และแบนเนอร์แสดงภาพสไลด์
แบบสำรวจครู survey_records, survey_educations, survey_subjects, survey_awards, survey_cefr, survey_hsk ข้อมูลโปรไฟล์ครู การศึกษา วิชาที่สอน ผลงาน รางวัล และระดับภาษาอังกฤษ/จีน
การอบรม (LMS) lms_courses, lms_lessons, lms_enrollments, lms_lesson_progress, lms_lesson_activity, lms_lesson_submissions หลักสูตร, บทเรียน, ข้อมูลเวลาดูวิดีโอ, การส่งงาน และการตรวจงานเพื่อรับใบประกาศ
ข้อสอบ (Quiz) lms_quizzes, lms_quiz_questions, lms_quiz_options, lms_quiz_attempts, lms_quiz_answers ข้อสอบก่อน/หลังเรียน คำถามและตัวเลือกประเมินผล คะแนนความก้าวหน้า

การตรวจสอบการจัดวาง UX/UI ระบบเดิม

ข้อสังเกตด้าน UX/UI: ระบบเดิมทำงานแยกส่วนอย่างชัดเจนและใช้โค้ด PHP แบบดั้งเดิม (Vanilla PHP) ที่มีการตกแต่งด้วย Tailwind CSS แบบฝังในบรรทัด (Inline Tailwind CSS) ในหน้าต่างๆ ซึ่งขาดความเป็นระเบียบและดูแลรักษายาก

เปรียบเทียบหน้าจอการใช้งาน (UX/UI mapping)

หน้าจอแบบเดิม (Legacy PHP)

  • fixed nav-bar: แถบเมนูดำทึบแบบคลาสสิก คอยสลับหน้าด้วยการเปลี่ยนไฟล์ PHP โดยตรง
  • dashboard.php: ใช้กราฟแสดงข้อมูลที่สร้างด้วย Chart.js มีแบนเนอร์แยกต่างหาก
  • form.php: ฟอร์มกรอกข้อมูลครูที่มีความยาวมากและการนำทางแบบเลื่อนยาว (Single Long Scroll)
  • report.php: ตารางแสดงข้อมูลบุคลากรทั้งหมดที่ดึงผ่าน DataTables.js ดั้งเดิม

แผนการอัปเกรดเป็น Blade ในเว็บใหม่

  • Laravel Layout Template: ครอบด้วย Blade layout สไตล์โมเดิร์น สวยงามและลื่นไหล
  • Interactive Dashboard Component: จัดการข้อมูลสถิติแยกการแสดงผลผ่านโครงสร้างแบบยืดหยุ่น
  • Multi-step Tabbed Form: แบ่งการกรอกประวัติครูเป็นขั้นๆ (ประวัติส่วนตัว -> การศึกษา -> ผลงาน) เพื่อลดความเหนื่อยล้าของสายตา
  • Optimized Directory List: ค้นหา กรองข้อมูลด้วย Alpine.js โหลดข้อมูลแบบรวดเร็ว

แผนภูมิการจับคู่ตารางข้อมูล (Schema Mapping Table)

การย้ายโครงสร้างข้อมูลจะใช้วิธีนำข้อมูลจากตารางดั้งเดิมเข้าสู่ฐานข้อมูล Laravel ใหม่ โดยมีโครงสร้างการแมปข้อมูลที่ตรงกันดังนี้:

ตารางเดิม (Legacy) ตารางใหม่ใน Laravel การเชื่อมโยงข้อมูล (Data Relationship)
users users เข้ากันได้โดยตรง (ใช้ ID ล็อคอินแบบภายนอกเหมือนเดิม)
survey_records survey_records (หรือสร้างตารางใหม่) สร้างตารางข้อมูลโปรไฟล์หลักของผู้สอน
survey_educations survey_educations ความสัมพันธ์ HasMany เชื่อมโยงผ่าน record_id
survey_subjects survey_subjects ความสัมพันธ์ HasMany เชื่อมโยงผ่าน record_id
survey_awards survey_awards ความสัมพันธ์ HasMany เชื่อมโยงผ่าน record_id
survey_cefr / survey_hsk survey_language_scores (หรือคงเดิม) ข้อมูลผลคะแนนวิชาภาษาอังกฤษและภาษาจีน
lms_* (ทั้งหมด 6 ตาราง) lms_* (เชื่อมผ่าน Eloquent Models) แมปตารางโครงสร้างหลักสูตร, สื่อวิดีโอ, และประวัติความก้าวหน้าการเรียน

ขั้นตอนการย้ายระบบ (Migration Execution Steps)

ขั้นตอนที่ 1: การสำรองข้อมูลระบบเก่า

ทำการดัมพ์ฐานข้อมูล e.cpn1.go.th ออกมาเป็นไฟล์ SQL จากนั้นทำการจัดทำโครงร่างฐานข้อมูล (Database Schema) ในรูปแบบ Laravel Migration สำหรับตารางฝั่งผู้ใช้งานและ LMS ทั้งหมด

ขั้นตอนที่ 2: การอัปโหลดรูปภาพคุณครู (Teachers Media)

ย้ายโฟลเดอร์ไฟล์อัปโหลดรูปภาพทั้งหมดจาก e/e.cpn1.go.th/uploads ไปเก็บไว้ใน storage/app/public/uploads ของระบบ Laravel และทำการรันคำสั่งเชื่อมโยงลิงก์มีเดียด้วย php artisan storage:link

ขั้นตอนที่ 3: พัฒนาตัวส่งผ่านข้อมูล (Laravel Data Migrator)

สร้างคำสั่งเฉพาะกิจใน Laravel เช่น app/Console/Commands/MigrateOldData.php เพื่อดึงข้อมูลจากตารางสำรวจประวัติครูและตาราง LMS เก่ามาแปลงและเติมลงในตารางใหม่ของ Laravel โดยอัตโนมัติ

ขั้นตอนที่ 4: การตั้งค่า Routing และสร้าง View หน้าแบบสำรวจ

ย้ายลอจิกหน้าสมัคร ล็อคอิน และหน้าการทำแบบประเมินและคอร์สเรียนไปใน Controller และเขียนหน้าการใช้งาน (Blade Views) โดยใช้สไตล์การดีไซน์ใหม่ที่เป็นมิตรต่อสายตา (Modern Dark Glassmorphism Theme)

การเชื่อมต่อระบบภายนอก (API & Assets Integration)

ระบบตรวจสอบสิทธิ์ภายนอก (External Salary API Integration)

ในระบบเดิม การเข้าสู่ระบบของครูจะต้องส่งรหัสไปตรวจสอบกับเซิร์ฟเวอร์ภายนอกที่:

API LOGIN URL: https://salary.cpn1.go.th/api/external/login
API KEY: EXTERNAL_LOGIN_API_KEY (เก็บในไฟล์ .env)

แผนการสร้าง Custom Guard ใน Laravel

ในระบบ Laravel ใหม่ เราจะนำระบบนี้มาแปลงเข้าสู่ระบบสถาปัตยกรรมของ Laravel อย่างเป็นทางการ:

1. เพิ่มการตั้งค่าใน .env:

เพิ่มคีย์ความปลอดภัย API_URL และ API_KEY ลงในระบบไฟล์สิ่งแวดล้อม (.env)

2. พัฒนา custom authentication provider:

สร้าง User Provider ใหม่ เช่น ExternalApiUserProvider ที่ทำหน้าที่ส่ง HTTP Request ไปตรวจสอบกับ API ด้วยระบบ Http::post() ของ Laravel แทนการใช้ Curl แบบดิบ

3. ซิงค์โปรไฟล์อัตโนมัติ:

เมื่อยืนยันสิทธิ์จาก API ภายนอกสำเร็จ ระบบจะนำค่าประวัติต่างๆ ของผู้ใช้มาใส่ลงในตาราง users ทันที เพื่อให้สิทธิ์ในการใช้ระบบอบรม LMS ได้อย่างราบรื่น
